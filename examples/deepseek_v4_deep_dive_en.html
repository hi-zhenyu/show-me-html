DeepSeek V4 Technical Report Deep Dive

This is not just a release summary. It is a structured reading of the DeepSeek V4 PDF across architecture, algorithms, data, pre-training, post-training, serving systems, and evaluation. The central thesis: V4 tries to make one-million-token context not merely possible, but deployable.

Reading verdict

DeepSeek V4 is best understood as a system-level efficiency project: CSA/HCA/SWA redesign attention, mHC stabilizes residual pathways, Muon supports trillion-scale MoE training, and OPD/QAT/serving infrastructure turn specialist capabilities into a deployable model.

0. Reading map

The report's skeleton

The 58-page report is organized around long-context efficiency, MoE training stability, post-training consolidation, and serving infrastructure. Benchmark tables are important, but they are downstream of that engineering stack.

Problem

One-million-token context is attention-bound

Vanilla attention makes FLOPs and KV cache growth the dominant bottleneck at extreme context length.

Architecture

Hybrid attention replaces one-size attention

CSA retrieves sparse compressed blocks, HCA keeps heavily compressed global memory, and SWA preserves local detail.

Training

Muon plus stability engineering

Muon updates most modules, while Anticipatory Routing and SwiGLU Clamping handle MoE loss spikes.

Post-train

Specialists first, OPD merge later

Math, code, agentic, and instruction specialists are trained first, then consolidated through multi-teacher on-policy distillation.

1. Architecture

The Transformer body remains; the critical paths change

V4 inherits DeepSeekMoE and MTP, then adds mHC, interleaved CSA/HCA attention, Muon, and routing/system changes for long-context efficiency.

Input

Tokens enter an architecture designed to maintain compressible state from the beginning.

mHC

Residual pathways are expanded and constrained for stable deep signal propagation.

SWA

Recent uncompressed KV entries preserve local fine-grained dependencies.

CSA/HCA

Long-range information is split between sparse compressed retrieval and dense heavy compression.

MoE

Every block uses MoE; early layers use hash routing; each token activates 6 routed experts.

MTP

Multi-token prediction depth remains 1, supporting training signal and serving throughput.

V4-Flash

Smaller activated footprint, efficiency-oriented

Scale

43 layers, hidden size 4096; 284B total parameters and 13B activated parameters.

Attention

First two layers use pure SWA; later layers interleave CSA/HCA. CSA m=4, top-k=512; HCA m'=128; SWA window=128.

MoE

1 shared expert plus 256 routed experts; 6 routed experts are activated per token.

V4-Pro

Larger capacity for knowledge and hard tasks

Scale

61 layers, hidden size 7168; 1.6T total parameters and 49B activated parameters.

Attention

First two layers use HCA; later layers interleave CSA/HCA. CSA m=4, top-k=1024; HCA m'=128; SWA window=128.

2. Algorithms and formulas

Why the long context becomes cheaper

The formulas below are rewritten for readability. They keep the core variable relationships while avoiding a direct reproduction of the paper's layout.

CSA

Compress, then select sparsely

Every m tokens become one compressed KV entry. A lightning indexer selects top-k compressed blocks for each query.

HCA

Compress more aggressively

Every m' tokens, where m' is much larger than m, become one entry. The model then performs dense MQA over that heavily compressed memory.

SWA

Do not compress local detail

A sliding-window branch keeps recent uncompressed KV entries so the compressed branches do not break local language modeling.

mHC residual update

X_{l+1} = B_l X_l + C_l F_l(A_l X_l)

Hyper-connections expand the residual stream; mHC constrains the residual mapping so deeper stacks remain numerically stable.

Doubly stochastic constraint

B_l in M = { M | M 1 = 1, 1^T M = 1^T, M >= 0 } ||B_l||_2 <= 1

Constraining B to the Birkhoff polytope makes the residual transformation non-expansive.

Sinkhorn projection

M^(0) = exp(B_tilde_l) M^(t) = T_row(T_col(M^(t-1))) B_l = M^(t_max), t_max = 20

Column and row normalization project the raw matrix onto the doubly stochastic manifold.

CSA compressed KV

C_a = H W_aKV, C_b = H W_bKV Z_a = H W_aZ, Z_b = H W_bZ C_i^comp = sum S_a,j * C_a,j + sum S_b,j * C_b,j

Two overlapping token windows are weighted and merged into one compressed entry.

CSA top-k sparse selection

c_t^Q = h_t W_DQ q_t^I = c_t^Q W_IUQ I_{t,s} = sum_h w_{t,h}^I ReLU(q_{t,h}^I K_s^IComp) C_t^sparse = { C_s^comp | I_{t,s} in Top-k(I_{t,:}) }

The indexer scores compressed history blocks and keeps only top-k for core attention.

HCA heavy compression

C = H W_KV, Z = H W_Z S = softmax_row(Z + B) C_i^comp = sum_j S_j * C_j

HCA uses a larger compression rate and skips sparse selection, trading detail for dense global memory.

Attention sink

s_{h,i,j} = exp(z_{h,i,j}) / (sum_k exp(z_{h,i,k}) + exp(z'_h))

The learnable sink lets attention mass be less than one instead of forcing every head to distribute all probability to tokens.

Muon update

G_t = grad_W L_t(W_{t-1}) M_t = mu M_{t-1} + G_t O'_t = HybridNewtonSchulz(mu M_t + G_t) O_t = O'_t sqrt(max(n,m)) gamma W_t = W_{t-1}(1 - eta lambda) - eta O_t

Most modules use matrix-level Muon updates; embeddings, heads, RMSNorm, and some mHC parameters stay on AdamW.

OPD objective

L_OPD(theta) = sum_i w_i KL(pi_theta || pi_Ei)

The student learns from multiple specialist teachers on trajectories sampled by the student itself.

Communication hiding

C / B <= V_comp / V_comm For V4-Pro: C / B <= 2d = 6144 FLOPs/Byte

Expert-parallel communication can be hidden when the compute-to-communication ratio satisfies the threshold.

3. Data and pre-training

The report discloses data directions, not mixture proportions

The PDF names major data categories and processing strategies, but does not provide exact token shares. This page therefore uses a data map rather than an invented pie chart.

Data construction

Pre-training data coverage

  • Starts from DeepSeek-V3 data and improves diversity, quality, and effective long context.
  • Filters batched auto-generated and templated web content to reduce model-collapse risk.
  • Keeps math and programming corpora as core components; adds agentic data during mid-training.
  • Expands multilingual data for long-tail cultural knowledge.
  • Prioritizes long documents such as scientific papers and technical reports.
  • Maintains a 128K vocabulary and inherits token-splitting and FIM strategies.

4. Post-training

Train specialists first, then consolidate with OPD

The main methodological substitution versus DeepSeek-V3.2 is replacing the mixed RL stage with On-Policy Distillation.

SFT

Each target domain first receives high-quality supervised fine-tuning.

GRPO RL

Math, code, agent, and instruction specialists are optimized with domain rewards.

GRM

Hard-to-verify tasks use rubric-guided RL data and a generative reward model.

Tool schema

DSML/XML-style tool calls reduce escaping failures and tool-call errors.

OPD

More than ten teachers transfer capabilities via full-vocabulary logit distillation.

QAT

MoE expert weights and the CSA indexer QK path adapt to FP4 deployment.

Reasoning modes

Non-think / High / Max

The three modes correspond to different RL settings, length penalties, context windows, and reasoning-token budgets.

Interleaved thinking

Persistent reasoning for tools

Tool-calling scenarios preserve reasoning across rounds; ordinary chat still discards previous reasoning after a new user message.

Quick instruction

Auxiliary tasks as special tokens

Search triggering, query generation, authority/domain classification, title generation, and URL reading decisions reuse the same KV cache.

5. Infrastructure

The architecture only pays off with matching systems

The report repeatedly ties model design to training and inference infrastructure. The efficiency claims depend on kernels, caching, scheduling, and deterministic serving behavior.

MegaMoE

Fine-grained expert pipeline

Dispatch, Linear-1, SwiGLU/FP8 cast, Linear-2, and Combine are decomposed into waves to overlap communication and computation. The report claims 1.50-1.73x common serving speedups and up to 1.96x in latency-sensitive settings.

TileLang

Fused kernels through DSL

TileLang reduces overhead from many small ATen operators and makes attention-variant experiments easier.

Determinism

Batch-invariant kernel library

MoE dispatch/combine and mHC small matrix operations are made deterministic to reduce batch-dependent output drift.

KV cache

Heterogeneous cache layout

CSA/HCA compressed KV, CSA indexer KV, SWA KV, and unfinished compression tails are managed separately.

On-disk prefix

Reusable shared prefixes

Compressed KV can be stored on disk; SWA cache uses full, periodic checkpoint, or zero-caching tradeoffs because it is roughly 8x larger.

DSec

Agent sandbox platform

The report describes an API gateway, host agents, cluster watcher, and 3FS storage for function calls, containers, microVMs, full VMs, and recoverable trajectories.

7. Caveats

How to read the claims skeptically

The report is information-rich, but it is still a preview technical report rather than a third-party reproduced study.

Architecture risk

Complex design stack

The authors acknowledge that many techniques remain in the architecture to reach extreme long-context efficiency.

Stability

Some fixes lack full theory

Anticipatory Routing and SwiGLU Clamping work empirically, but the underlying mechanisms remain open.

Evaluation

Many numbers are internal

Table 1 uses an internal unified framework, while agent/search/white-collar/coding-agent evaluations rely heavily on internal harnesses or human evaluation.

Data

Mixture details are not public

The report names categories and processing strategies, but not exact data shares or contamination tests.

Serving

Real cost depends on deployment

Lower FLOPs and KV cache do not directly determine user-facing price or latency.

Product

Training format is not API contract

Think/tool schemas in the report should not be read as final guarantees about public API behavior.
