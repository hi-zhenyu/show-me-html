这版不再只是摘要，而是按论文结构把架构、算法、训练数据、预训练、后训练、推理系统和评测结果串起来。核心判断：V4 的主战场是把百万 token 上下文从“能跑”压到“能常规部署”。

阅读结论

DeepSeek-V4 的技术路线可以概括为：用 CSA/HCA/SWA 改造注意力，用 mHC 改造残差路径，用 Muon 和稳定性策略支撑万亿 MoE 训练，再用 OPD、FP4 QAT、工具/Agent 数据和百万上下文基础设施完成后训练与服务。

0. Reading map

先建立整篇报告的骨架

报告共 58 页，主线不是单个 benchmark，而是长上下文效率、MoE 训练稳定性、后训练合并和服务系统的联动。

Problem

百万上下文卡在注意力成本

vanilla attention 的计算和 KV cache 在超长上下文中成为瓶颈，也限制 test-time scaling 和长周期 Agent。

Architecture

混合注意力替代单一注意力

CSA 负责压缩+稀疏检索，HCA 负责重压缩+稠密注意力，SWA 保住短程局部依赖。

Training

Muon + 稳定性工程

预训练用 Muon 更新多数模块；MoE loss spike 用 Anticipatory Routing 和 SwiGLU Clamping 压住。

Post-train

专家先强化，再蒸馏统一

数学、代码、Agent、指令等专家模型经过 SFT/RL 后，通过多教师 OPD 合入一个统一模型。

1. Architecture

架构：Transformer 主体不变，关键路径全部重做

V4 继承 DeepSeekMoE 和 MTP，但新增 mHC、CSA/HCA 混合注意力、Muon 优化器，并调整 MoE 路由与系统并行方式。

Input

token 进入 embedding，超长上下文从一开始就按可压缩状态组织。

mHC

用受约束的动态连接增强残差路径，重点是稳定深层信号传播。

SWA

保留最近窗口的未压缩 KV，解决本块内与近邻 token 的细节依赖。

CSA/HCA

长程信息分别走压缩稀疏注意力与重压缩稠密注意力。

MoE

所有 block 使用 MoE；早期 3 层使用 Hash routing；每 token 激活 6 个 routed experts。

MTP

多 token 预测深度为 1，继续作为训练信号和推理吞吐优化基础。

V4-Flash

小激活量、强调性价比

规模

43 层，hidden 4096；总参数 284B，激活 13B。

注意力

前两层纯 SWA，之后 CSA/HCA 交错；CSA m=4、top-k=512；HCA m'=128；window=128。

MoE

1 shared expert + 256 routed experts；每 token 激活 6 个；expert hidden 2048。

V4-Pro

更大知识容量、更强难题表现

规模

61 层，hidden 7168；总参数 1.6T，激活 49B。

注意力

前两层 HCA，之后 CSA/HCA 交错；CSA m=4、top-k=1024；HCA m'=128；window=128。

MoE

1 shared expert + 384 routed experts；每 token 激活 6 个；expert hidden 3072。

2. Algorithms & formulas

算法与核心公式：为什么能压长上下文

这里把论文中的公式改写成可读版本，保留变量关系而不复制排版细节。重点是看每个公式解决什么工程问题。

CSA

压缩 + 稀疏选择

每 m 个 token 压成一个 KV entry，再用 lightning indexer 给 query 选择 top-k 压缩块。适合“从很长历史里检索少量关键块”。

HCA

更激进压缩 + 稠密读

每 m' 个 token 压成一个 entry，m' 远大于 m；不做 sparse top-k，而是对重压缩后的序列做 MQA。

SWA

近邻信息不压缩

CSA/HCA 只能看之前完整压缩块，所以额外保留最近 nwin 个未压缩 KV，避免局部语言建模被压坏。

mHC 的稳定性约束

M = { B ∈ R^(n×n) | B·1 = 1, 1^T·B = 1^T, B ≥ 0 } ||B_l||₂ ≤ 1

残差映射 B 被约束到双随机矩阵流形：行列和都为 1，元素非负。这样谱范数不超过 1，减少深层堆叠时的信号放大。

Sinkhorn-Knopp 投影

M^(0) = exp(B~_l) M^(t) = T_row(T_col(M^(t-1))) B_l = M^(t_max), t_max = 20

先把 unconstrained 参数指数化为正，再交替做列归一化和行归一化，得到满足 mHC 约束的 B。

CSA 压缩 KV

C_a = H·W_aKV, C_b = H·W_bKV Z_a = H·W_aZ, Z_b = H·W_bZ C_i^comp = Σ S_a,j ⊙ C_a,j + Σ S_b,j ⊙ C_b,j

CSA 使用两路相邻/重叠窗口，通过 softmax 权重把 2m 个候选压成一个 block entry，实际序列长度压到约 1/m。

Lightning indexer 选择 top-k

c_t^Q = h_t·W_DQ q_t^I = c_t^Q·W_IUQ I_{t,s} = Σ_h w^I_{t,h} · ReLU(q^I_{t,h} · K^Icomp_s) C_t^sparse = { C_s^comp | I_{t,s} ∈ Top-k(I_{t,:}) }

对 query 和历史压缩块打分，只保留 top-k 块参与核心注意力，这是 CSA 在 1M 场景省 FLOPs 的关键。

HCA 重压缩

C = H·W_KV, Z = H·W_Z S_{m'i:m'(i+1)-1} = softmax_row(Z + B) C_i^comp = Σ_{j=m'i}^{m'(i+1)-1} S_j ⊙ C_j

HCA 不使用重叠压缩，也不做 sparse selector；它用更大的 m' 把序列压得更短，再稠密读取。

Attention sink

s_{h,i,j} = exp(z_{h,i,j}) / (Σ_k exp(z_{h,i,k}) + exp(z'_h))

分母里多加可学习 sink logit，让某个 head 的总注意力质量不必等于 1，甚至可以接近 0，减少无意义强制分配。

Muon 更新

G_t = ∇_W L_t(W_{t-1}) M_t = μ M_{t-1} + G_t O'_t = HybridNewtonSchulz(μM_t + G_t) O_t = O'_t · sqrt(max(n,m)) · γ W_t = W_{t-1}(1 - ηλ) - ηO_t

Muon 对大多数模块做矩阵级更新和正交化；embedding、prediction head、RMSNorm 等仍用 AdamW。

Hybrid Newton-Schulz

M_k = aM_{k-1} + b(M_{k-1}M_{k-1}^T)M_{k-1} + c(M_{k-1}M_{k-1}^T)^2M_{k-1}

前 8 次迭代用激进系数快速靠近奇异值 1，后 2 次用稳定系数收敛；目标是近似得到 U·V^T。

OPD 多教师目标

L_OPD(θ) = Σ_i w_i · D_KL(π_θ || π_Ei)

学生模型在自己采样出的轨迹上，向多个领域专家的输出分布做 reverse-KL 对齐，避免直接权重合并的退化。

通信可隐藏条件

C / B ≤ V_comp / V_comm DeepSeek-V4-Pro: C / B ≤ 2d = 6144 FLOPs/Byte

MoE expert parallelism 的关键不是无限堆带宽，而是计算/通信比例满足阈值后，通信能被计算覆盖。

3. Data & pre-training

数据与预训练流程：报告披露了方向，没有披露配比

这里要特别谨慎：PDF 给出了数据类型和处理策略，但没有公开各类数据占比，所以不能画“真实数据配比饼图”。

Data construction

预训练数据覆盖面

  • 在 DeepSeek-V3 数据基础上继续提升多样性、质量和有效长上下文。
  • Web 数据过滤批量自动生成和模板化内容，降低 model collapse 风险。
  • 数学与代码仍是核心；mid-training 阶段加入 agentic data 来增强编码能力。
  • 多语言语料扩大，目标是覆盖长尾文化知识。
  • 长文档特别强调科学论文、技术报告等学术价值材料。
  • tokenizer 维持 128K 词表，继承 token-splitting 与 FIM；新增上下文构造特殊 token。

4. Post-training

后训练：先造专家，再用 OPD 合并能力

报告把 DeepSeek-V3.2 的 mixed RL 阶段替换成 On-Policy Distillation，把多个领域专家的行为蒸馏进统一模型。

SFT

每个目标领域先用高质量领域数据建立基础能力。

GRPO RL

数学、代码、Agent、指令等专家分别用领域 reward 信号强化。

GRM

硬验证任务用 rubric-guided RL data 和生成式奖励模型评价轨迹。

Tool schema

引入 DSML/XML 风格工具调用，减少转义失败和工具调用错误。

OPD

十多个教师模型，full-vocabulary logits distillation。

QAT

MoE expert weights 与 CSA indexer QK path 做 FP4 量化感知训练。

Reasoning modes

Non-think / High / Max

三档推理模式来自不同 RL 配置：长度惩罚、上下文窗口和输出 reasoning token 长度不同。Max 还通过系统提示推动更彻底的推理。

Interleaved thinking

工具场景保留推理历史

在 tool-calling 场景，V4 保留跨轮 reasoning 内容以支持长周期 Agent；普通聊天仍在新用户消息后丢弃旧 reasoning。

Quick instruction

把辅助任务变成特殊 token

搜索触发、标题、query、authority、domain、URL 读取判断等由同一模型复用 KV cache 完成，避免额外小模型重复 prefill。

5. Infrastructure

基础设施：模型结构的收益依赖系统实现

V4 的复杂架构如果没有训练和推理系统配合，很难兑现效率收益。报告在这里给了很多工程细节。

MegaMoE

专家并行的细粒度流水

把 Dispatch、Linear-1、SwiGLU/FP8 cast、Linear-2、Combine 拆成 wave，让通信、计算、结果回传并行。报告称一般推理 1.50-1.73×，延迟敏感场景最高 1.96×。

TileLang

用 DSL 写融合 kernel

大量细粒度 ATen operator 会拖慢训练和推理，TileLang 用 host codegen 和 fused kernels 降低调用开销，也便于快速试验注意力变体。

Determinism

batch-invariant 与确定性 kernel

MoE dispatch、combine、mHC 小矩阵乘法都做了确定性处理，服务和训练中减少 batch 变化导致的非确定输出。

KV cache

异构 KV cache layout

把 CSA/HCA compressed KV、CSA indexer KV、SWA KV、未完成压缩尾部状态分开管理，突破传统 PagedAttention 的同构假设。

On-disk prefix

共享 prefix 复用

CSA/HCA 压缩 KV 可落盘复用；SWA 因体积约为压缩 KV 的 8 倍，提供 full caching、periodic checkpointing、zero caching 三种取舍。

DSec

Agent 沙箱平台

报告描述 Rust API gateway、per-host agent、cluster watcher 和 3FS 存储，支持函数调用、容器、microVM、fullVM，并记录 trajectory 以便恢复和复现。

7. Caveats

限制与我会如何谨慎使用这些结论

这份报告信息量很大，但仍是 preview 技术报告，不是第三方复现论文。下面这些点不该被省略。

Architecture risk

结构非常复杂

论文自己也承认，为追求极端长上下文效率保留了许多技巧，未来要把架构蒸馏得更优雅。

Stability

稳定性技巧还缺理论解释

Anticipatory Routing 和 SwiGLU Clamping 有效，但底层机制仍没有充分理论理解。

Evaluation

很多评测是内部口径

Table 1 是统一内部框架；Agent、搜索、白领任务、代码 Agent 等也大量依赖内部 harness 或人评。

Data

数据配比没有公开

报告给了数据类别和处理策略，但没有公开各类 token 占比、去重标准、污染检测细节和合成数据比例。

Serving

真实成本取决于部署

FLOPs/KV cache 比例下降不等于最终价格或延迟；硬件、kernel、缓存命中率、批处理和 API 定价都会影响体验。

Product

训练格式不等于 API 承诺

报告展示 think/tool schema 和 reasoning 模式，但最终开放接口如何暴露、是否展示 reasoning 内容，应以实际服务文档为准。
